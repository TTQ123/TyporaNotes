### 2.CSS常用的选择器

```css
1.元素选择器（根据标签名选中指定的元素） 例如:*{样式名：样式值;} 选择所有元素  h1{样式名：样式值;} 选中所有h1的标签 问题：有点麻烦
2.id选择器（根据标签名的id选中指定的元素）
例如：#red{样式名：样式值;}      问题:不可以重复使用
3.类选择器（根据标签的class属性选中指定的元素） 优点：可以为一个标签设置多个class名称 <p class="blue red abc"></p>
例如：.blue{样式名：样式值;}       问题：可以重复使用
4.通配选择器 *{样式名：样式值;} 选择页面的所有标签
```

### 9.选择器的权重（样式优先级）

```css
⭐⭐⭐内联(行内)样式 > ID 选择器 > 类选择器 = 属性选择器 = 伪类选择器 > 标签选择器 = 伪元素选择器
选择器优先级计算===>>>
内联样式 1,0,0,0
id选择器 1,0,0
类和伪类选择器 0,0,1,0
元素选择器 0,0,0,1
通配选择器 0,0,0,0
继承的样式 没有优先级
选择器写的越具体，它的优先级一般越高
1.比较优先级时，需要将所有的选择器进行相加计算，最后优先级越高，越优先显示。
2.选择器的累加不会超过其最大的数量级，类选择器再高也不会超过id选择器
3.如果优先级计算后相同，此时优先使用代码靠下的样式，相当于被覆盖了
4.通配选择器的优先级为0
内联>id选择器>类和伪类选择器>元素选择器
5.继承的样式没有优先级
6.css可以在某一个样式后加 !important 变成最高的优先级 
在开发中这个要慎用！！
```

#### 1.文档流是什么

```css
1.文档流

关于文档流需要记住:文档流是网页最底下的一层，是网页的基础。

分为在文档流和不在文档流

	块元素
		1.在页面中独占一行
		2.默认宽度是父元素的宽度（会把父元素撑满）
		3.默认高度是被内容撑开（子元素撑开的内容）
	行内元素
		1.行内元素只占自身的大小
		2.行内元素在页面中会自左向右排列，如果一行中不能容纳所有的内容，会自动换到第二行自左向右排列
		3.行内元素的宽高默认都是被内容撑开
```

#### 2.⭐⭐⭐盒子模型⭐⭐⭐

```css
1.css将页面中所有的元素都设置了一个矩形的盒子
2.对页面的布局，就是将不同的盒子摆放到自己想要的位置
3.盒子的部分由几个部分组成
	一丶内容区（content）
		width:设置内容区宽度
		height:设置内容区高度
			
	二丶边框（border）盒子的边边
		边框的大小影响了盒子模型的大小
		要设置边框必须设置三个样式
			1.边框的宽度 border-width
				默认值是3px左右
				border-width可以指定四个方向的边框的宽度
					border-width:10px 20px 30px 40px
					四个值的顺序（顺时针）:上右下左
					三个值的顺序:上 左右 下
					俩个值的顺序:上下 左右
					一个值的顺序:四个方向一样
				border-xxx-width:top right bottom left
					用来单独指定某一个边的宽度
			2.边框的颜色 border-color
				省略不写则自动使用color的颜色值
				和border-width规则一样
			3.边框的样式 border-style
				border-style默认值是none表示没有边框 也有四个方向
				solid 表示实线
				dotted 点状的虚线
				dashed 虚线
				double 双线
				
	####border简写属性可以同时设置边框的所有样式，不要求顺序  border:20px red solid
	border-xxx:指定一边的样式
	@@开发常用写法 
		border:10px red solid;
		border-right:none;

	三丶内边距（padding）内容区和盒子边框之间的距离
		内边距的设置会影响盒子的大小
		背景颜色会延伸到内边距上
		共有四个方向的内边距
		padding-top
		padding-right
		padding-bottom
		padding-left
		！！盒子一共的可见框大小，由内容区 内边距和边框共同决定
		计算盒子大小时要将三个区域加到一起计算
		padding是内边距的简写属性 同时设置四个方向 
		四个值的顺序（顺时针）:上右下左
					三个值的顺序:上 左右 下
					俩个值的顺序:上下 左右
					一个值的顺序:四个方向一样
					
	四丶外边距（margin）决定了俩个盒子碰撞在一起时所在的位置
	外边距不会影响盒子可见框的大小，但是会影响盒子的位置
	一共有四个方向的内边距
		margin-top
		上外边距，设置一个正值元素会向下移动
		margin-right
		默认情况下设置它不会有什么效果
		margin-bottom
		下外边距，设置一个正值元素会向右移动
		margin-left
		左外边距，设置一个正值元素会向右移动
		margin也可以设置负值，负值会向反方向移动

		元素一般是自左向右排序
		一般会靠左上角，所以默认情况下如果我们设置的是左和上外边距则会移动自身
		如果我们设置的是右和下外边距则会移动其它元素
	magrin也是简写属性，和padding一样
	##magrin会影响到盒子的实际占用空间
	左右magrin都为0则为居中  magrin：0 auto;
```

#### 3.⭐⭐⭐水平方向布局

```css
元素在水平方向的布局由以下几个属性共同决定
		magrin-left
		border-left
		padding-left
		width
		padding-right
		border-right
		magin-right
		一个元素在其父元素中，水平布局必须满足以下条件
♥  左外边距+左边框+左内边距+内容宽度+右外边距+右边框+右内边距=其父元素内容区的宽度
！！以上等式必须成立，如果相加的结果不相等，则称为过度约束，则等式会自动调整
    调整的情况:如果这7个值中没有为auto(0)的情况，则浏览器会自动调整magin-right使等式满足 
  ！！！！如果有auto，这7个值有3个值可以设置为auto
  width
  magrin-left
  magrin-right
  如果某个值为auto，则会自动调整那个auto的值以使等式成立
  width的默认值是auto，magrin默认值是auto
1.如果将宽度和一个外边距auto，则宽度会调整到最大，设置为auto的外边距会自动为0
2.如果三个值都为auto，则外边距都是0，宽度会达到最大
3.如果俩个外边距都为auto，宽度设置了固定字值，则会将外边距设置为相同的值
利用第三点这个特性可以给一个元素设置其在父元素中水平居中
例:width="100px";
magin:0 auto;
还有一种情况下，等式左边设置的值大于右边的值，浏览器会把magrin-right设置为负值
```

#### 4.垂直方向的布局

```css
默认情况下，父元素的高度会被内容撑开，设置了多少也就是多少
子元素是在父元素的内容区中排列的，如果子元素的大小超过了父元素，则子元素会从父元素中溢出
使用overflow属性设置父元素解决子元素的溢出问题
overflow
	可选值：
		visible 默认值，子元素会从父元素中溢出，在元素外部显示
		hidden 内容会被修剪，并且其余内容是不可见的。
		scroll 内容会被修剪，但是浏览器会显示滚动条以便查看其余的内容
		auto  如果内容被修剪，则浏览器会显示滚动条以便查看其余的内容
	    inherit	规定应该从父元素继承 overflow 属性的值
```

#### 5.外边距的折叠

```css
相邻的垂直方向外边距会发生重叠的现象
	1.兄弟元素
		-兄弟间的相邻垂直外边距会取俩者之间的较大者（最大值）
		特殊情况 --相邻的外边距一正一负，则会取俩者的和
		--如果相邻的外边距都是负值，则取俩者中绝对值较大的
		--兄弟元素之间的外边距重叠，对开发时是有利的，不用处理
	2.父子元素
		--父子元素间相邻外边距，子元素的会传给父元素（上外边距）
		--父子外边距的折叠会影响页面的布局，要进行处理
		低端处理方案：不让相邻   不用外边距
```

## 4.进阶

### 1.浮动

```css
通过浮动使元素向着父元素的左侧或右侧移动
	通过float来设置浮动
	可选值:
		默认值 none，元素不移动
		left 元素向左移动
		right 元素向右移动
	元素设置浮动后（不是none），水平布局的等式不需要强制成立
	元素设置浮动后（不是none），会完全从文档流中脱离，不在占用文档流的位置
	所以元素下边还在文档流的元素会自动向上移动
	
浮动的特点:
	1.浮动元素会完全脱离文档流，不在占据文档流当中的位置
	2.设置浮动以后元素会向父元素的左侧或者右侧移动
	3.浮动元素默认不会从父元素中移除
	4.！！浮动元素向左或向右移动时，不会超过它前边的其它浮动元素
	5.如果浮动元素的上方是一个没有浮动的元素，则浮动元素无法上移
	6.浮动元素不会超过它上边的浮动的兄弟元素，最多就是和它一样高，同一高度（此场景在屏幕缩小时经常出现）
总结:浮动的主要作用就是让页面中的元素可以水平排列,通过浮动可以制作一些水平方向的布局

浮动的高级特点:
1.浮动元素不会盖住元素，文字会自动环绕在浮动元素周围
	可以利用这个效果设置文字环绕图片的效果
2.元素设置浮动以后，会从文档流中脱离，从文档流脱离后，会有一些其它的特点
	脱离文档流的特点:
		块元素
			1.块元素不在独占一行
			2.脱离文档流后，块元素的宽度和高度默认都被内容撑开
		行内元素
			行内元素脱离文档流会变成文档流，特点和块元素一样，脱离文档流以后,不需要在区分块和行内元素了
			块元素的特点
				1.在页面中独占一行
				2.默认宽度是父元素的宽度（会把父元素撑满）
				3.默认高度是被内容撑开（子元素撑开的内容）
```

### 2.高度坍塌(元素设置完浮动以后)和BFC

```css
高度塌陷的问题:
	在浮动布局，父元素的高度默认被子元素的内容区撑开
		当子元素浮动后，脱离文档流，将会无法撑开父元素，父元素因此高度坍塌
		父元素高度丢失后，其下的元素将会上移，导致页面布局混乱
BFC(块级格式化环境)
	-BFC是css中一个隐含（不是用字段开启）的属性，可以为一个元素开启一个BFC
		-开启BFC后该元素会变成一个独立的布局区域
	开启后BFC后的特点
		1.开启BFC的元素不会被浮动元素给覆盖
		2.开启BFC的元素子元素和父元素的外边距不会重叠
		3.开启BFC的元素可以包含浮动的子元素
我们可以通过一些特殊的方式来开启BFC元素（开启后会有副作用）
	1.设置元素的浮动（不推荐）
	2.将元素设置为行内块元素（不推荐）
	3.将元素的overflow设置为一个非visible的值（推荐）
		-为元素设置overflow hidden 开启其BFC 以使其可以包含浮动元素
	4.设置元素的绝对定位
```

### 3.clear

```css
如果我们不希望某个元素因为其它元素浮动的原因改变了位置
我们可以为该元素设置clear属性来清除浮动元素的影响
clear清除浮动元素对当前元素的影响（浮动的元素还是会浮动）
可选值:
	--left:清除左边浮动元素的影响
	--right:清除右边浮动元素的影响
	--both:清除俩边对元素影响最大的那边
原理:设置清除浮动后，浏览器会自动为元素添加一个上外边距
--以使其不受其它元素的影响
```

### 4.使用伪类after来清除高度塌陷的方案（最终方案）

```css
例如在div后加一个伪类元素来解决高度塌陷
after元素结尾处
div::after{
content:'';//设置伪类
display:block;//将伪类（行内元素）转换成块元素
clear:both;//利用clear清除影响
//既不浮动也不脱离文档流
}
```

### 5.clearfix（解决相邻外边距的重叠）

```css
.clearfix 这里是类名噢小老弟
.clearfix::before元素开始前
.clearfix::after元素结束处

完美方案  俩个都处理

            content: '';
            display: table;
            clear: both;
        }

一、.clearfix::before+ content: '';+display: table;解决外边距重叠
二、.clearfix::after+ content: '';+display: table;+clear: both;解决高度塌陷
```

## 5.定位

### 1.相对定位

```css
高级的布局手段（解决了盒子模型和浮动的缺点）通过定位可以任意的移动页面中的一个元素，不影响其它元素的布局
定位 position
	可选值:
		--static 默认值，元素是静止的，没有开启定位
		--relative  相对定位
		--absolute  绝对定位
		--fixen 固定定位
		--sticky 粘滞定位
相对定位的特点:
   1.元素开启相对定位以后，如果不设置偏移量元素不会发生任何改变
   2.相对定位是参照于元素在文档流中的位置来进行定位的（定位位置，相当于是参照自己原来的位置来定位，就是没有开始移动前的那个点）
	3.相对定位会提升元素的层级，比浮动元素等元素级别高
	4.相对定位不会使元素脱离文档流
	5.相对定位不会改变元素的性质，行内元素还是行内元素，块元素还是块元素
	6.相对定位相当于灵魂出窍，灵魂移动了，肉体还占据着之前的位置（还占据着移动前的位置）
	偏移量（offset）
	top
		-定位元素和定位位置上边的距离
		top越大元素越往下走
	bottom
		-定位元素和定位位置下边的距离
		bottom越大元素越往上走
两个元素共同控制垂直方向的定位，通常情况下只使用一个元素来控制
	
	left
		-定位元素和定位位置左边的距离
		left越大元素越往右走
	right
		-定位元素和定位位置右边的距离
		right越大元素越往左走
```

### 2.绝对定位

```css
绝对定位
	-当元素的position属性设置为absolute时，则开启了绝对定位
	-绝对定位的特点:
		1.开启绝对定位后，如果不设置偏移量元素的位置不会发生变化
		2.开启绝对定位后，元素会从文档流中脱离
		3.绝对定位会改变元素的性质，行内变成块，块的宽度被内容撑开
		4.绝对定位会使元素提高一个层级
		5.绝对定位元素是相对于其包含块进行定位的
		(包含块 contaning block)
			--正常情况下:
				--包含块就是当前元素最近的祖先块元素
				<div><span><em>hello</em></span></div>
				em的包含块就是div（span不是块元素）  span的包含块也是div
			--开启绝对定位的包含块:
				--！！包含块就是离它最近的开启了定位的祖先元素
				（只要position的值不是static）
				--如果所有的祖先元素都没有开启定位，则根元素(html)就是它的包含块	
```

### 3.固定定位

```css
固定定位
	--固定定位也是一种绝对定位，大部分特点和绝对定位一样
	1. 坐标原点固定，始终相对于浏览器窗口定位。 
	2. 位置固定，移动滚动条时，固定定位的元素位置仍然不变。（参考一些页面旁边的小广告）
```

### 4.粘滞定位

```css
position:sticky;
类类似于相对定位
不同的是可以在元素到达某个位置时将其固定
```
